The creative problem

TinyFish can't access your filesystem.
So we gave it a URL instead.

TinyFish runs in its own isolated browser session. It can't read local files. But it needs to upload a filled PDF to BizFile+. Our solution: use GitHub as the bridge.

OpenAI + pypdf

Fill PDF locally

GPT-4o maps extracted fields to 45 AcroForm names. pypdf writes the PDF.

→
GitHub API

Upload to GitHub

Filled PDF pushed to repo via Contents API. Returns a raw URL.

→
raw.githubusercontent

Public URL

PDF now accessible at a URL any browser can reach.

→
TinyFish Agent

Download + upload

Agent downloads PDF from GitHub, uploads it to BizFile+ form.

Why this matters: TinyFish's goal includes the URL: "download the PDF from {pdf_url} then upload it using the file input". The agent navigates the form, fills fields, attaches the PDF, and stops at the review page. GitHub becomes the filesystem bridge between our backend and TinyFish's isolated browser.
